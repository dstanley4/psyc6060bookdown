15.1 Internet data

We will use a data set where students are nested within 100 classes. We will look at an overall correlation, then the correlation within each group.

These data are from Chapter 2 of the excellent Joop Hox Multilevel Modeling book. The read_csv command can load data from a website. The data are loaded from the URL:

https://jeroen.github.io/multilevel/data/chapter2/popular2.csv

We load the data with the command:

library(tidyverse)
library(janitor)

my_data <- read_csv("https://jeroen.github.io/multilevel/data/chapter2/popular2.csv")

my_data <- my_data %>%
  remove_empty("rows") %>%
  remove_empty("cols") %>%
  clean_names()

15.3 Creating factors

Looking at the data reveals that the column class might be considered a categorical variable/factor. However, if we convert class to a factor we loose the ability to use operations like greater than (>) or less than (<) on the class column – because factors are categorical only (i.e., the group numbers cease to be numbers and become labels). So in this unusual case we will opt to keep group as a numeric value - because of our later plans for it (i.e., to extract groups based on group numbers). Therefore, we do not use as.factor to create a factor version of the class column.

15.4 Single correlation

We are interested in obtaining the correlation between extraversion and popularity. To do we need to extract the specific columns from the analytic data set. We can do this with notation illustrated below – the dollar sign indicates we want a column from the data. This notation is used in following code block.

analytic_data$extrav    #This obtains the extrav  column from the data set
analytic_data$popular   #This obtains the popular column from the data set

We can use this notation in the cor.test command below. Notice that you get extensive output – including the confidence interval for the correlation.

cor.test(x = analytic_data$extrav,
         y = analytic_data$popular)
## 
## 	Pearson's product-moment correlation
## 
## data:  analytic_data$extrav and analytic_data$popular
## t = 14.877, df = 1998, p-value < 2.2e-16
## alternative hypothesis: true correlation is not equal to 0
## 95 percent confidence interval:
##  0.2757798 0.3547147
## sample estimates:
##       cor 
## 0.3157935

Thus, you would report: r = .32, 95% CI [.28, .35], p < .001.

Notice that in reporting the correlation this way we emphasize the effect size (r) and confidence interval by reporting them first. The p-value should be reported last because the information it provides is of quite limited value.

head(analytic_data)

You can quickly plot the data with the ggplot command as per below. Although this graph is not APA style you can easily make it APA with the commands from other handouts.

my_scatter <- ggplot(data = analytic_data,
                     mapping = aes(x = extrav, y = popular)) +
               geom_point() 

print(my_scatter)

A regression line can easily be added to graph using the geom_smooth function. In this function, method = “lm” indicates a linear model should be used to create the line (i.e., a linear regression should be used). As well, se = FALSE indicates the confidence interval around the regression line should not be displayed.

my_scatter <- ggplot(data = analytic_data,
                     mapping = aes(x = extrav, y = popular)) +
               geom_point() +
               geom_smooth(method = "lm",
                           se = FALSE,
                           color = "black")

print(my_scatter)
## `geom_smooth()` using formula 'y ~ x'

Try making the graph again - but this time use: se = TRUE to see the confidence interval around the regression line.

Note you can see all the options for geom_smooth with the help command below:

?geom_smooth

analytic_data_class_is_5 <- filter(analytic_data, class == 5)

15.6.2 Multiple subgroups

15.6.2.1 Background

Before we can calculate correlation for subgroups we need to know how R stores/handles information

This next bit is about how R works - not how you would typically run the analysis. Going into detail about how R works will help you to understand what we do later in this exercise when we encounter subgroups. With R, you can store the result of any analysis in a variable. In this case, you can store the correlation information in a variable we will call my_cor_result.

my_cor_result <- cor.test(x = analytic_data$extrav,
                          y = analytic_data$popular)

If you type my_cor_result you get all the information:

my_cor_result
## 
## 	Pearson's product-moment correlation
## 
## data:  analytic_data$extrav and analytic_data$popular
## t = 14.877, df = 1998, p-value < 2.2e-16
## alternative hypothesis: true correlation is not equal to 0
## 95 percent confidence interval:
##  0.2757798 0.3547147
## sample estimates:
##       cor 
## 0.3157935

But you can see the structure of the resulting information with the str() function.

str(my_cor_result)
## List of 9
##  $ statistic  : Named num 14.9
##   ..- attr(*, "names")= chr "t"
##  $ parameter  : Named int 1998
##   ..- attr(*, "names")= chr "df"
##  $ p.value    : num 1.48e-47
##  $ estimate   : Named num 0.316

my_cor_result$estimate

You can use this information to access just the correlation in a later section when we deal with subgroup analyses.

15.6.2.2 Implementation

If you’re interested in getting a correlation for 100 classes, there is any easy way to do so. We begin by using the group_by command from the tidyverse package.

analytic_data_grouped_by_class <- group_by(analytic_data, class)

Then you calculate the correlation for each group:

cor_by_group <- summarise(analytic_data_grouped_by_class, 
                          cor_for_class = cor.test(x = extrav,
                                                   y = popular)$estimate)
## `summarise()` ungrouping output (override with `.groups` argument)

You can then print the correlation for the first few groups with the print command. Notice that the result for class 5 (i.e., r = .64) is the same as when we did analyses specifically for group 5 in the previous section.

print(cor_by_group)

You can print all the rows with:

print(as.data.frame(cor_by_group))

As previously noted, it’s a good idea to inspect the scatter plot for each correlation you calculate. In this case, that means you want a plot of the correlation for each group. To do so, we get the subset of the groups you want (e.g., just groups 15 to 30) using the filter command from the tidyverse package. This command creates a new data set with just the desired groups. In the example below we create a data set that contains just groups 15 to 30.

analytic_data_groups_15_to_30 <- filter(analytic_data_grouped_by_class, 
                                        class >= 15 & class <= 30)

The scatter plot for those 15 groups is generated by using the ggplot2 package with the commands below. Don’t forget to install the tidyverse package (which contains ggplot2) if you haven’t used it before.

library(tidyverse)

my_scatter <- ggplot(data = analytic_data_groups_15_to_30,
                     mapping = aes(x = extrav, y = popular)) +
               geom_point() +
               facet_wrap(~class)

print(my_scatter)

You can easily add the regression line for these groups. Notice how the regression line does not extend beyond the data points. This is ideal behavior for the software because regression predictions are not valid beyond the range of the data.

library(tidyverse)

my_scatter <- ggplot(data = analytic_data_groups_15_to_30,
                     mapping = aes(x = extrav, y = popular)) +
               geom_point() + 
               geom_smooth(method = "lm",
                           se = FALSE,
                           color = "black") +
               facet_wrap(~class)

print(my_scatter)
## `geom_smooth()` using formula 'y ~ x'

library(tidyverse)
my_scatter <- ggplot(data = analytic_data_groups_15_to_30,

15.7.1 Within a data set

15.7.1.1 Non-overlapping correlations

We will compare the correlation to between (a1, c1) to the correlation between (e1, o1) with cocor. In this case, because neither of the variables in the first correlation (a1, c1) are in the second correlation (e1, o1) we refer to this as a non-overlapping correlation comparison.

The cocor command will provide a lot of output. We are most interested in the last part of the output corresponding to Zou (2007). But we also want to examine the first part of the output which will show us the two original correlations (a1, c1) = 0.0276, and (e1, o1) = -0.1002. As well it also shows us the difference between them, Difference: r.jk - r.hm = 0.1278.
